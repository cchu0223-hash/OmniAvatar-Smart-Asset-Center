# PRD：OmniAvatar AI对口型功能集成 v5.7.5.1

> **版本说明**：本文档是 PRD5.7.5（含 rev1、rev2 全部修订）的整合版，以产品评审为导向重新组织，内容与 5.7.5-rev2 完全对齐，**不新增功能**。

---

## 一、背景与目标

### 1.1 迭代背景

V5.7.2 版本完成了对口型核心流程（落地页上传 → 解析 → 编辑 → 生成）的首次上线并验证可行。本次迭代（V5.7.5）在此基础上重点完成两件事：

1. **专业剪辑页集成对口型**：用户无需离开剪辑环境，即可对轨道上的视频片段直接发起对口型生成；
2. **交互体验细化**：对生成完成后的操作路径、任务管理体验、引导气泡等细节进行完善。

### 1.2 核心目标

| # | 目标 | 说明 |
|---|------|------|
| 1 | 剪辑内完整对口型工作流 | 在专业剪辑页内嵌完整的「解析 → 编辑 → 生成 → 替换」流程 |
| 2 | 统一任务管理 | 落地页与剪辑页发起的对口型任务，统一在「我的」Tab 呈现 |
| 3 | 双向任务同步 | 任意页面发起的任务，均可在两端实时查看进度 |
| 4 | 降低使用门槛 | 首次使用引导气泡 + 右键一键触发，减少路径摩擦 |

### 1.3 目标用户

- **短剧出海团队**：需要将剪辑好的视频快速生成多语种对口型版本；
- **跨境电商运营**：需要将产品宣传视频适配英、日、韩等多个市场；
- **首次体验新用户**：通过推荐案例了解对口型效果，并快速完成首次生成。

### 1.4 支持语言（8 种，与 V5.7.2 一致）

ZH（中文）/ EN（英语）/ JA（日语）/ RU（俄语）/ KO（韩语）/ AR（阿拉伯语）/ ES（西班牙语）/ YUE（粤语）

> **不支持**：法语（FR）、德语（DE）——需求文档与产品实现均不得扩展至以上两种语言。

### 1.5 不在本次范围内

- 批量上传（队列式处理多个视频）
- 字幕导出（SRT / VTT 格式）
- 阿拉伯语 RTL 布局完整适配
- 生成完成全局 Toast 通知
- 处理中任务取消（算力不退还）
- 任务关键词搜索与状态筛选
- 算力充值页面、登录注册流程（本 PRD 仅定义触发入口）


## 三、需求详情

### 3.1 对口型落地页（`/lipsync`）

#### 3.1.1 导航栏

导航栏固定在页面顶部，不随滚动消失。包含以下元素（从左到右）：

- **Logo**：点击跳转首页（`/`）
- **「AI素材生成」**：跳转 `/`
- **「对口型」**：当前页，高亮显示
- **「专业剪辑」**：跳转 `/professional-edit`
- **算力余额**：实时展示当前用户算力，每次生成任务提交后主动刷新一次
- **用户头像**（已登录）/ **「登录」按钮**（未登录）

#### 3.1.2 Hero 区（idle 态）

**展示内容**：
- 主标题：两段式打字机动效，先出现「讯飞智作」，再出现「，AI对口型一步到位」，整行单行展示，不折行
- 副标题：「多语种真实对口型视频，让您的视频全球发声。」
- 品牌示例视频：自动播放、静音、循环，展示对口型前后对比效果（不提供「做同款」按钮）
- 上传入口：支持拖拽上传和点击上传

**idle 态下 Tab 区（推荐 / 我的）正常可见**，Hero 区与 Tab 区上下排布。

#### 3.1.3 视频上传（uploading 态）

**触发方式**：用户拖拽或点击上传入口选择文件。

**登录校验**：
- 已登录 → 直接进入上传流程
- 未登录 → 弹出登录弹窗；登录成功后自动继续上传

**上传规格**：
- 支持格式：MP4、MOV
- 时长限制：≤ 5 分钟
- 文件大小：≤ 500MB

**上传中展示**：
- Tab 区收起隐藏
- 工作区显示上传进度面板：文件名 + 进度条 + 百分比数字

⚠️ **边界条件**：

| 情形 | 系统行为 |
|------|---------|
| 文件格式不支持（非 MP4/MOV） | 上传前拦截，展示「格式不支持，请上传 MP4 或 MOV 格式」，保持 idle 态 |
| 文件超过 500MB | 上传前拦截，展示「文件过大，请上传 500MB 以内的视频」，保持 idle 态 |
| 视频时长超过 5 分钟 | 上传前拦截，展示「视频时长超限，请上传 5 分钟以内的视频」，保持 idle 态 |
| 上传过程中网络中断 | 展示「上传超时，请检查网络后重试」+「重新上传」按钮 |
| 上传失败（服务端错误） | 展示具体错误原因 +「重新上传」按钮，回到上传入口态 |

#### 3.1.4 语音解析（parsing 态）

**触发时机**：视频上传成功后自动触发，无需用户操作。

**展示内容**：
- Tab 区保持收起隐藏
- 工作区显示圆形渐变进度环 + 中心百分比 + 「视频解析中…」文案
- 系统自动识别视频源语言（支持中文、英文）

⚠️ **边界条件**：

| 情形 | 系统行为 |
|------|---------|
| 解析超时 | 展示「解析超时，请重试」+「重新上传」按钮，允许用户重新选择文件 |
| 解析失败（无法识别语音） | 展示「未能识别视频中的语音内容」+「重新上传」按钮 |
| 源语言为不支持的语言 | 展示「暂不支持该语言的对口型，当前支持中文和英文原声」，提供「重新上传」按钮 |

#### 3.1.5 台词编辑区（editing 态）

**进入方式**：解析完成后自动进入；或通过「做同款」/「重新编辑」直接进入（台词预填）。

**展示内容**：

**目标语言选择**
- 下拉选择器，支持 8 种目标语言
- 切换后 1.4 秒动画过渡，台词内容自动替换为翻译结果
- 源语言（自动识别）展示为只读标签，不可修改

**台词列表**
- 按讲话人（讲话人 1、讲话人 2…，按首次出现顺序命名）+ 时间段分组展示
- 每条台词默认为只读展示，点击后进入编辑态：
  - 展示多行输入框 + 「保存」「取消」按钮
  - `Escape` 键 = 取消（放弃修改）
  - `Enter` 键（非换行）= 保存
  - 台词内容与解析原稿无差异时，「保存」按钮禁用
  - 点击保存后按钮短暂进入「保存中」状态，完成后回到只读态

**台词字数限制**
- 单段台词上限：200 字
- 总字数上限：由产品配置决定（默认 2000 字）
- **单段超限提示**：仅在「已保存后的只读态」该段落下方展示「该片段字数超出 N，请分段或精简」，编辑草稿中不提示
- **总字数超限提示**：顶部黄色警告条显示，同时「生成视频」按钮禁用

**做同款台词差距提示**
- 通过「做同款」进入 editing 后，用户编辑台词时，若修改幅度与原始台词相比差距过大（字符相似度 < 60%），该段落展示「视频生成效果可能欠佳」提示（仅提醒，不阻止保存）

**底部操作栏（固定在编辑区底部，不随台词列表滚动）**
- 预计算力消耗展示
- 「返回」按钮：清空当前 editing 态，回到 idle（Hero + Tab 区重新显示）
- 「生成视频」按钮

**「生成视频」禁用条件**（满足任一即禁用）：
- 总字数超限
- 任一台词段处于「保存中」状态
- 存在未关闭的台词编辑框（即有段落处于编辑态）

⚠️ **边界条件**：

| 情形 | 系统行为 |
|------|---------|
| 算力不足 | 弹出算力不足提示弹窗，含「去充值」跳转链接；弹窗关闭后保持在 editing 态 |
| 切换目标语言后翻译失败 | 展示「翻译失败，请重试」提示，保持上一次成功的语言内容 |
| 用户在翻译动画过渡中再次切换语言 | 取消上一次切换动画，立即响应最新选择 |

#### 3.1.6 生成提交（generating 态）

**触发时机**：用户点击「生成视频」，且算力充足。

**系统行为（按顺序）**：
1. 调用生成接口，接口同步返回 `taskId`
2. 前端立即将该任务以「处理中 / 进度 0%」状态插入「我的」Tab 列表（乐观 UI，不等待后端确认）
3. 工作区切换至生成提交成功面板，展示「任务已提交，可在『我的』查看进度」
4. 「我的」Tab 顶部展示提示条：「任务已提交，正在处理中」
5. 页面自动切换至「我的」Tab，工作区回到 idle（Tab 区重新显示）

**生成提交面板**展示：
- 提交成功提示文案
- 「新建任务」按钮：点击清空状态，工作区回到 idle

> **无全局 Toast**：生成任务提交及完成后，均不弹出全局 Toast 通知。任务完成后，「我的」Tab 中的任务卡片状态自动更新为「成功」，用户需主动切换至「我的」Tab 查看结果。

⚠️ **边界条件**：

| 情形 | 系统行为 |
|------|---------|
| 生成接口调用失败（网络/服务端错误） | 展示「提交失败，请重试」，保持在 editing 态，不插入任务列表 |
| 用户在 generating 态刷新页面 | 状态机重置为 idle，任务已提交的任务仍在「我的」Tab 中保留（后端持久化） |

#### 3.1.7 推荐 Tab

**访问权限**：未登录用户可正常浏览，仅点击「做同款」时触发登录。

**搜索功能**：
- 支持关键词搜索
- 预置推荐词（硬编码在前端）：短剧 / 个人演讲 / 教育培训 / 广告营销 / 品牌营销

**视频卡片**：
- 展示封面图、视频标题、场景分类标签、目标语言标签
- 封面图懒加载

**点击视频卡片 → 对比播放弹窗**：
- 左侧：原视频播放器；右侧：对口型后视频播放器
- 两个播放器**独立控制**，互不联动（播放/暂停/进度条/音量/全屏各自独立）
- 底部「做同款」按钮：
  - 已登录 → 关闭弹窗，直接进入 editing 态（源视频 = 示例视频，台词预填，目标语言 = 该推荐视频的目标语言）
  - 未登录 → 弹出登录弹窗；登录成功后自动进入 editing 态

⚠️ **边界条件**：

| 情形 | 系统行为 |
|------|---------|
| 搜索关键词无匹配结果 | 展示「暂无相关案例」空态提示 |
| 推荐视频加载失败 | 展示卡片占位符 + 刷新提示 |

#### 3.1.8 我的 Tab

**访问权限**：
- 未登录 → 点击「我的」Tab 触发登录弹窗
- 登录成功后 → 停留在「我的」Tab，加载任务列表

**任务来源**：展示所有对口型任务，包括落地页发起的任务和专业剪辑页发起的任务，**来源对用户不可见**，统一在列表中呈现。

**任务卡片字段**：

| 字段 | 说明 |
|------|------|
| 视频缩略图 | 视频首帧自动截取 |
| 任务名称 | 视频文件名 |
| 目标语种 | 生成时选择的目标语言 |
| 视频时长 | **仅「合成成功」状态展示**（如 `1m14s`）；其他状态不展示 |
| 创作时间 | YYYY-MM-DD HH:mm |
| 状态标签 | 成功 / 处理中 / 失败 |

**点击缩略图** → 弹出单视频预览弹窗（含播放控件）

**操作按钮（按状态）**：

| 任务状态 | 可用操作 |
|---------|---------|
| 成功 | 下载、重新编辑、专业剪辑（跳转 ProEditor）、删除 |
| 处理中 | 进度条 + 百分比（后端轮询，间隔 ≤ 3s）；其余按钮禁用；**不提供取消** |
| 失败 | 重新编辑、删除 |

**「重新编辑」行为**：点击后重新提交一个新任务，进入 editing 态，源视频 = 该任务原视频，台词 = 该任务最后保存的台词内容，可修改；原任务保留在列表中。

**列表加载**：滚动到底部自动加载更多（每次 20 条），无搜索/筛选功能。

**进度轮询规则**：处理中任务前端定时轮询后端进度接口（间隔 3 秒），任务状态变为「成功」或「失败」时自动停止轮询。

⚠️ **边界条件**：

| 情形 | 系统行为 |
|------|---------|
| 任务列表为空 | 展示空态图 +「暂无对口型任务，上传视频开始体验」 |
| 删除任务 | 弹出「确认删除？删除后无法恢复」确认弹窗，确认后从列表移除 |
| 下载链接过期 | 展示「下载链接已过期，请重新生成」，提供「重新编辑」入口 |
| 网络恢复后进度轮询 | 断线期间暂停轮询，网络恢复后自动重启轮询 |

### 3.2 专业剪辑页对口型功能

#### 3.2.1 入口设计

**入口 1：右键菜单（主入口）**

- 用户右键点击 V1 视频轨道上的任意片段
- 上下文菜单中新增「AI对口型」选项，强调色展示，与其他选项用分隔线区分
- 点击后：
  1. 左侧面板自动切换至对口型面板
  2. 左侧导航栏底部**动态出现「对口型」图标**（首次触发时才出现，出现后持续可见）
  3. 直接使用该轨道片段的视频进入解析（`parsing`）阶段，无需重新上传

> **生成完成后右键菜单变化**（`done` 态）：原「AI对口型」选项替换为两项：
> - 「查看生成任务」→ 打开对口型面板，展示结果预览态
> - 「一键替换轨道」→ 直接执行替换逻辑

**入口 2：左侧导航栏对口型图标**

- 图标**不默认存在**，仅在用户首次通过右键触发对口型后才出现在导航栏底部
- 出现后保持可见（即使关闭对口型面板或切换至其他面板）
- 点击图标：
  - 已选中 V1 轨道视频片段 → 直接进入 `parsing` 阶段
  - 未选中 → 面板展示空态「请先选中轨道上的视频片段」

**入口 3：首次使用引导气泡**

- **触发时机**：用户首次在 V1 轨道插入视频片段（拖入或粘贴）
- **展示位置**：视频片段右上角，箭头指向片段
- **内容**：💡「试试 AI 对口型，右键即可开始」+「知道了」按钮
- **消失逻辑**（满足任一即消失）：
  - 3 秒后自动淡出
  - 用户点击「知道了」按钮
  - 用户右键该片段时立即关闭
- **展示次数**：仅展示一次，通过浏览器本地存储标记，刷新页面后不再重复展示

⚠️ **边界条件**：

| 情形 | 系统行为 |
|------|---------|
| 右键的不是 V1 轨道（如 A1 音频轨） | 右键菜单不展示「AI对口型」选项 |
| 右键时对口型面板已处于 `generating` 或 `done` 态 | 见上文「生成完成后右键菜单变化」；`generating` 态仍展示「AI对口型」，点击后切换至对口型面板查看当前进度 |
| 导航图标出现后用户关闭整个剪辑项目后重新打开 | 图标状态随 LocalStorage 记录持久化，重新打开后图标仍存在 |

#### 3.2.2 对口型面板（各阶段）

面板固定宽度 288px，位于左侧面板区，与素材面板、配音面板平级切换，按当前阶段展示不同内容。

---

**空态（idle）**

- 居中展示图标 + 文案「请先选中轨道上的视频片段」
- 用户在 V1 轨道选中片段后，自动进入 `parsing` 阶段

---

**解析阶段（parsing）**

- 顶部展示素材信息：视频文件名、轨道名、时长
- 圆形渐变进度环（紫 → 青）+ 中心百分比
- 文案「视频解析中…」

⚠️ 解析失败：展示具体原因 +「重新解析」按钮（重新发起解析，不需要重新上传）

---

**台词编辑阶段（editing）**

- 顶部：解析成功提示横幅（青色背景 + ✓ 图标）
- 目标语言下拉选择（8 种，切换后 1.4s 翻译动画过渡）
- 台词列表（按讲话人 + 时间段分组）：
  - 默认只读展示
  - **点击台词** → 进入编辑态，展示输入框 + 「保存」「取消」
  - 台词偏差限制：若修改后内容与解析原稿的相似度低于阈值（归一化 Levenshtein 偏差 ≥ 35%），展示警告且**不可保存**（需将台词调整至合理范围才能保存）
  - 台词与解析稿无差异时，「保存」按钮禁用
- 底部固定栏：
  - 预计算力消耗
  - 「开始AI对口型」按钮

> **剪辑页台词偏差限制与落地页的区别**：剪辑页台词编辑启用修改幅度阈值拦截（偏差 ≥ 35% 不可保存）；落地页台词编辑**不启用**此拦截，仅展示「视频生成效果可能欠佳」提示。

⚠️ **边界条件**：

| 情形 | 系统行为 |
|------|---------|
| 算力不足点击「开始AI对口型」 | 弹出算力不足提示弹窗，含「去充值」链接 |
| 台词偏差超过 35% 时点击保存 | 保存按钮禁用，展示警告文案，引导用户减小修改幅度 |

---

**生成中阶段（generating）**

- 面板展示：「对口型生成中…」+ 当前进度百分比
- 「开始对口型」按钮变为禁用态，显示「生成中…」
- 面板内容不可操作
- **用户可自由切换至其他面板**继续剪辑工作，不影响生成进度
- V1 轨道对应片段叠加渐变进度条（见 3.2.3）

---

**生成完成阶段（done）**

- 面板顶部：「生成完成 ✓」标识
- 16:9 视频播放区：支持播放/暂停、下载、全屏
- 底部操作按钮：
  - 「一键替换轨道」（主按钮，渐变紫蓝色）
  - 「重新编辑」（次按钮）

**「重新编辑」行为**：

无论当前任务处于何种状态，点击「重新编辑」均重新提交一个新任务，重新进入 editing 态，原台词预填可修改；原任务保留在「我的」Tab。

#### 3.2.3 轨道进度叠加层

V1 轨道被选中的视频片段，在对口型生成期间展示进度叠加层：

**生成中**：
- 半透明渐变进度条叠加在片段上方（紫 → 青，透明度 70%）
- 进度条宽度 = 片段宽度 × 进度百分比
- 进度条上方居中展示「对口型中 XX%」文字标签
- 进度更新有过渡动画（0.35s）

**生成完成**：
- 进度条变为「100% 生成完成」蓝青渐变覆盖层
- **持续展示，不自动消失**，直至用户主动执行替换

**多片段场景**：
- 对口型状态以**单个 clip 为粒度**绑定
- 右键哪个 V1 片段，即对该片段发起对口型，各片段状态独立互不影响
- 同一时间可有多个片段处于不同阶段

#### 3.2.4 一键替换轨道

**触发方式**（两处入口等效）：
1. 对口型面板 `done` 态「一键替换轨道」按钮
2. 右键菜单（`done` 态）「一键替换轨道」选项

**执行逻辑**：
- 轨道叠加层（overlay）消失
- 片段缩略图更新为对口型视频的首帧
- 片段视频源更新为生成后的视频 URL
- 片段在时间轴上的**位置和时长不变**（原子操作，不新增轨道）
- 替换完成后，对口型面板状态机重置为 `idle`

⚠️ **边界条件**：

| 情形 | 系统行为 |
|------|---------|
| 替换过程中网络中断 | 展示替换失败提示，overlay 保持显示，可重试 |
| 执行替换后撤销（Ctrl+Z） | 撤销为轨道还原，但对口型面板状态不随撤销变化（仍为 idle） |

#### 3.2.5 任务双向同步

**同步规则**：
- 剪辑页点击「开始AI对口型」后，后端接口同步返回 `taskId`
- 前端将该任务写入落地页任务列表（调用统一后端 API）
- 落地页「我的」Tab 与剪辑页各自独立轮询同一 `taskId` 的进度
- 任务不区分来源（`landing` / `editor`），落地页展示与落地页自身发起的任务完全一致
