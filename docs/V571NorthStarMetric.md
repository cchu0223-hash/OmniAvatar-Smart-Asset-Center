## 一、生成量

**核心公式：** 日/周/月生成任务数

| 维度 | 使用事件 | 拆解方式 |
|------|---------|---------|
| 总生成量 | ZZWB800600 + ZZWB800610 + ZZWB513100 + ZZWB513300 | 按时间聚合 |
| 图片 vs 视频 | `gen_type` 属性 | 分类对比趋势 |
| 模型偏好 | `model` 属性 | 各模型使用占比 |
| 入口来源 | 800600+800610（首页）vs 513100/513300（剪辑页） | 判断哪个入口贡献更高 |
| 做同款带来的生成 | 801200 → 800600/800610 同 session 链路 | 推荐驱动的生成量占比 |
| 算力消耗 | `credit_cost` 属性 | 人均算力消耗，预测付费转化时机 |

## 二、留存率

**核心公式：** N日后回访并再次触发生成的用户占比

| 维度 | 使用事件 | 拆解方式 |
|------|---------|---------|
| 复访生成 | ZZWB801600（再次生成）+ ZZWB801500（重新编辑后生成） | 老用户回访创作频次 |
| 参数回填行为 | ZZWB802000（ActionDock回填）+ ZZWB801500 | 用户对历史任务的依赖程度，越高留存越好 |
| 素材沉淀 | ZZWB801700（同步至智能素材）+ ZZWB513500（使用到轨道） | 用户在产品内"存资产"的深度，资产越多流失成本越高 |

## 三、商业转化

**核心公式：** 算力耗尽用户 → 购买行为转化率（结合现有 ZZWB100010/100020）

| 维度 | 使用事件 | 拆解方式 |
|------|---------|---------|
| 高价值用户识别 | ZZWB800700 + ZZWB800710 `credit_cost` 累计 + ZZWB513200/513400 | 算力消耗速度快的用户优先推付费 |
| 视频生成转化价值 | ZZWB800610/ZZWB513300 `credit_cost` | 视频单次消耗更高，是付费触发器 |
| 从生成到购买漏斗 | 800600/800610 → 800800/800810（失败/余额不足）→ ZZWB100020（下单） | 还原完整付费路径 |

## 四、创作完成率

**核心公式：** 生成成功数 / 生成触发数

| 维度 | 使用事件 | 拆解方式 |
|------|---------|---------|
| 首页完成率 | (ZZWB800700 + ZZWB800710) / (ZZWB800600 + ZZWB800610) | 按 `model` 分组，找出哪个模型失败率高 |
| 剪辑页图片完成率 | ZZWB513200 / ZZWB513100 | 对比首页同类型完成率，判断入口质量差异 |
| 剪辑页视频完成率 | ZZWB513400 / ZZWB513300 | 视频耗时长，失败成本高，需重点监控 |
| 失败原因分布 | ZZWB800800 + ZZWB800810 `fail_reason` | 区分余额不足 vs 系统错误，指导产品/技术优化方向 |
| 生成耗时分布 | `gen_duration_sec` 属性（800700/800710/513200/513400） | P50/P90耗时，影响用户感知完成率 |
| 素材利用率 | ZZWB513500 / (ZZWB513200 + ZZWB513400) | 生成后被实际用到轨道的比例，衡量生成内容的质量 |
